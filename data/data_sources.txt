Population data source
----------------------
Primary source:
U.S. Census Bureau, Population Division
City and Town Population Totals: 2020-2024
Vintage 2024 / SUB-IP-EST2024 city and town population estimates
https://www.census.gov/data/tables/time-series/demo/popest/2020s-total-cities-and-towns.html

How this project currently uses it:
The map reads population data from data/population.js. That file is generated by python/generate_population.py from official Census subcounty estimates CSV files stored in python/raw_data. Location coordinates come from the official Census 2024 National Places Gazetteer stored at python/raw_data/2024_Gaz_place_national.zip. The script joins those files by Census place GEOID and keeps incorporated places and consolidated cities.

Downloaded source files:
https://www2.census.gov/programs-surveys/popest/datasets/2000-2010/intercensal/cities/sub-est00int.csv
https://www2.census.gov/programs-surveys/popest/datasets/2010-2020/intercensal/cities/sub-est2020int.csv
https://www2.census.gov/programs-surveys/popest/datasets/2020-2024/cities/totals/sub-est2024.csv
https://www2.census.gov/geo/docs/maps-data/data/gazetteer/2024_Gazetteer/2024_Gaz_place_national.zip

Coverage note:
The generated map-ready population file contains exact Census city/place population years 2000 through 2024. As of May 4, 2026, the checked Census FTP path for 2020-2025 city/town/place totals did not contain a city/town totals CSV, so this project does not fabricate 2025 city population values.

Air quality data source
-----------------------
Primary source:
U.S. Environmental Protection Agency (EPA)
Outdoor Air Quality Data / AirData
https://www.epa.gov/outdoor-air-quality-data

Related EPA visualization/source page:
AirData Multiyear Tile Plot
https://www.epa.gov/outdoor-air-quality-data/airdata-multiyear-tile-plot

How this project currently uses it:
The map reads air quality data from data/historical.js and data/sites.js. Those files were generated from the EPA AirData annual monitor CSV files stored in python/raw_data and cleaned files in python/cleaned_data. The generated records include monitor location, pollutant averages, and computed AQI-style values used for map coloring.

Wind data source
----------------
Primary source:
NOAA National Centers for Environmental Information (NCEI)
Integrated Surface Database (ISD) Lite
https://www.ncei.noaa.gov/products/land-based-station/integrated-surface-database

How this project currently uses it:
The map reads wind data from data/wind.js. That file is generated by python/download_wind.py
from NOAA ISD Lite hourly station observations cached in python/wind_raw/. The script selects
~280 geographically representative US continental stations (one per 20x14 lat/lon grid cell),
downloads their hourly data for years 2000-2025, computes annual mean wind velocity (u/v
components via vector averaging of hourly wind direction and speed), then IDW-interpolates onto
a 60x40 lat/lon grid. The browser uses bilinear interpolation on that grid to get a real wind
vector for each particle position. Fallback synthetic flow is used outside continental US bounds
or before wind.js has been generated.

Downloaded source files:
https://www.ncei.noaa.gov/pub/data/noaa/isd-history.csv  (station registry)
https://www.ncei.noaa.gov/pub/data/noaa/isd-lite/{year}/{USAF}-{WBAN}-{year}.gz  (per station/year)
